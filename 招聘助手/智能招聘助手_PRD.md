# 智能招聘助手 PRD

版本：v1.1  
日期：2026-05-18  
适用范围：BOSS 直聘企业版沟通工作台、招聘助手浏览器插件、Web 管理后台、后端服务、钉钉机器人。

## 1. 背景

当前招聘工作中，HR 需要在 BOSS 直聘沟通列表中反复打开候选人、查看在线简历、判断与岗位 JD 的匹配度、索要简历、整理推荐表，并在钉钉群中同步招聘进展。现有系统已经具备插件采集、Web 后台、历史数据、钉钉推送和问答 Agent 的雏形，但仍存在以下问题：

- 候选人信息识别不稳定，部分记录姓名或关键字段为空。
- 新岗位 JD 抓取失败或抓取到聊天内容，岗位要求库无法持续沉淀。
- 匹配度达到阈值的候选人没有稳定触发求简历动作。
- Web 后台功能集中在单页，模块边界不清晰，筛选和账号隔离能力不足。
- 钉钉推送、Excel、问答 Agent 与权限体系尚未形成完整闭环。

## 2. 产品目标

建设一个面向 HR 的智能招聘操作系统，实现从候选人识别、简历采集、JD 入库、匹配评分、索要简历、推荐汇总、钉钉问答到数据治理的自动化闭环。

核心目标：

- 全自动处理当前列表全部可见候选人。
- 准确采集候选人顶部摘要和完整在线简历。
- 所有岗位要求库中缺失的岗位必须尝试抓取 JD 并入库。
- 以大模型为主进行 JD 与简历匹配评分。
- 匹配度达到 40% 以上的候选人必须索要简历。
- 按岗位、账号、状态、匹配度对候选人进行归类。
- 支持 Web 后台、插件、钉钉机器人三端协同。
- 支持账号权限隔离、操作风控和审计证据。

## 3. 用户角色

### 3.1 管理员

权限：

- 查看全部招聘账号和全部候选人数据。
- 创建、审核、禁用后台用户。
- 审核普通用户与招聘账号的绑定关系。
- 配置大模型、钉钉、自动化策略、白名单、安全参数。
- 维护岗位要求库。
- 清理历史数据。
- 配置是否保存原始简历全文和展示脱敏规则。

### 3.2 普通用户

权限：

- 首次登录后选择或绑定招聘账号，等待管理员审核。
- 仅查看被授权招聘账号的数据。
- 使用插件执行当前账号候选人采集任务。
- 查看候选人、推荐结果、岗位库和推送记录。
- 发起问答 Agent 查询，但查询范围受 Web 后台权限限制。

### 3.3 钉钉群成员

权限：

- 在钉钉群中向机器人提问。
- 默认可查询全部招聘数据。
- 后续可扩展为按钉钉用户身份映射后台权限。

## 4. 使用场景

### 4.1 候选人批量处理

HR 打开 BOSS 直聘沟通页面，点击插件一键执行，系统自动处理当前列表全部可见候选人。

### 4.2 新岗位首次出现

系统发现候选人的沟通职位在岗位要求库中不存在，自动点击“沟通职位”打开岗位详情，抓取工作内容、工作要求、任职资格等 JD 正文并入库。

### 4.3 推荐候选人推进

候选人与 JD 匹配度达到 40% 以上，系统自动点击“求简历”；若找不到按钮，发送兜底消息“方便发一份您的简历过来吗？”。

### 4.4 简历接收

候选人主动发送附件简历，系统自动点击同意或接收，标记为“有附件简历”，继续进入匹配和推荐流程。

### 4.5 招聘数据同步

系统按“上次推送后到本次推送前”的增量范围推送钉钉消息和 Excel 推荐表。

### 4.6 问答 Agent

用户在 Web 后台或钉钉群提问，例如“前端开发有哪些高匹配候选人”“为什么某候选人不推荐”，Agent 基于历史数据、JD 和推荐报告回答。

## 5. 端到端流程

1. 任务启动：插件一键、Web 后台远程下发或定时自动执行。
2. 范围确定：默认处理当前列表全部可见候选人。
3. 打开候选人：依次点击候选人列表项，进入沟通界面。
4. 基础识别：采集姓名、沟通职位、学历、经验、薪资、当前公司、账号、来源、最近沟通时间。
5. 入库校验：候选人至少具备“姓名 + 沟通职位 + 学历或经验任一项”才允许入库。
6. JD 检查：查询岗位要求库是否存在该岗位。
7. JD 抓取：若不存在，点击“沟通职位”抓取岗位详情；成功入库，失败标记“待补录”。
8. 简历采集：点击在线简历，采集顶部摘要和完整简历；随机化滚动深度。
9. 关闭详情：采集完成后按“继续沟通 > 打招呼 > ×号”优先级关闭详情页。
10. 匹配评分：以大模型为主，基于 JD 和完整简历输出匹配度、推荐等级、原因和证据。
11. 降级评分：大模型不可用、超时、额度不足时降级本地规则。
12. 求简历：匹配度 40% 以上优先点击求简历按钮，找不到则发送兜底消息。
13. 附件接收：有附件简历请求时自动同意或接收。
14. 数据归档：按岗位、账号、状态、匹配度归类。
15. 推送汇总：按增量范围推送钉钉正文和 Excel。
16. 问答查询：Web 后台和钉钉机器人可基于历史数据回答问题。

## 6. 功能需求

### 6.1 浏览器插件

#### 6.1.1 一键执行任务

- 插件必须保留“一键开始执行任务”按钮。
- 点击后处理当前列表全部可见候选人。
- 支持后台远程下发任务。
- 支持定时自动执行，且只在配置的工作日工作时间内运行。

#### 6.1.2 候选人识别

必须识别：

- 姓名
- 沟通职位
- 学历
- 经验
- 薪资或期望薪资
- 当前公司
- 最近沟通时间
- 数据来源
- 招聘账号

最低入库标准：

- 姓名 + 沟通职位 + 学历或经验任一项。

低于标准：

- 不入库。
- 写运行日志。
- 可在 Web 后台“异常日志”中查看。

#### 6.1.3 在线简历采集

- 必须采集候选人顶部摘要信息。
- 必须点击在线简历并采集完整简历。
- 完整简历采集包括工作经历、项目经历、教育经历、技能、证书、自我描述。
- 滚动策略采用随机化深度，不同候选人不完全一致。
- 采集时不得混入其他候选人、列表、导航、聊天记录。
- 采集完成后必须关闭详情页，优先级为继续沟通、打招呼、×号。

#### 6.1.4 附件简历处理

- 出现“对方想发送附件简历给您，您是否同意”等提示时，自动点击同意或接收。
- 标记为“有附件简历”。
- 记录附件接收动作证据。

#### 6.1.5 求简历动作

- 匹配度 40% 以上必须索要简历。
- 优先点击“求简历”按钮。
- 找不到按钮时发送“方便发一份您的简历过来吗？”。
- 同候选人同岗位不重复索要。
- 同候选人不同岗位可重新索要。
- 同候选人同岗位超过 7 天仍未收到简历，可再次发送一次兜底消息。

#### 6.1.6 操作风控

必须支持：

- 随机延迟。
- 每日操作上限。
- 每小时操作上限。
- 滚动模式模拟。
- 停留时间模拟。
- 鼠标轨迹模拟。
- 风控关键词识别。
- 异常页面识别。
- 按账号配置不同节奏策略。
- 疑似风控自动暂停。

异常处理：

- 风控或页面结构变化：暂停任务并提醒人工处理。
- 单个候选人异常：跳过并继续下一个。
- 按钮找不到：执行兜底消息或写日志。

### 6.2 岗位要求库

#### 6.2.1 自动抓取

- 系统首次遇到岗位要求库中不存在的岗位时，必须点击“沟通职位”打开岗位详情。
- 抓取字段必须优先来自岗位详情弹窗中的：
  - 工作内容
  - 工作职责
  - 岗位职责
  - 职位描述
  - 职位介绍
  - 工作要求
  - 任职要求
  - 任职资格
  - 岗位要求
  - 职位要求
- 不得把候选人聊天内容、候选人简历内容或候选人列表误存为 JD。

#### 6.2.2 手动维护

支持：

- 新增 JD。
- 编辑 JD。
- 删除 JD。
- 重复岗位自动合并。
- 按岗位、账号、状态筛选。
- JD 更新后按管理员配置重新匹配历史候选人。

#### 6.2.3 待补录

- JD 抓取失败时，岗位进入“待补录”状态。
- Web 后台岗位库显示待补录岗位。
- 每日钉钉汇总提醒待补录岗位。

### 6.3 匹配评分

#### 6.3.1 评分方式

- 大模型评分为主。
- 输入为岗位 JD + 候选人完整简历。
- 输出匹配度、推荐等级、匹配原因、风险点、缺失项、证据摘录。
- 大模型不可用时，自动降级本地规则评分。

#### 6.3.2 推荐等级

- 40%-59%：推荐。
- 60%-79%：非常推荐。
- 80% 及以上：强烈推荐。

#### 6.3.3 推荐动作

- 40% 以上候选人必须索要简历。
- 钉钉推送包含所有 40% 以上候选人，包括无附件简历但已索要简历者。

#### 6.3.4 报告要求

推荐报告必须包含：

- 姓名
- 岗位
- 学历
- 经验
- 薪资
- 匹配度
- 推荐等级
- 匹配原因
- 风险点
- 不推荐依据
- JD 逐条对照
- 缺失项
- 下一步沟通建议
- 大模型原始评分依据
- 可追溯证据摘录

### 6.4 候选人数据管理

#### 6.4.1 去重规则

- 平台唯一 ID 优先。
- 没有平台唯一 ID 时，按姓名 + 岗位去重。
- 不区分账号去重。

#### 6.4.2 归类维度

必须支持：

- 按岗位归档。
- 按账号归档。
- 按处理状态归档。
- 按匹配度排序。

#### 6.4.3 状态流转

默认状态：

- 新增
- 已查看
- 已索要简历
- 已收到简历
- 推荐面试
- 不推荐

#### 6.4.4 不推荐处理

- 默认不自动婉拒。
- 管理员可配置是否自动婉拒。
- 管理员可配置温和版或直接版婉拒话术。

### 6.5 Web 管理后台

#### 6.5.1 模块结构

后台按功能分为三组：

工作台：

- 总览
- 候选人
- 岗位库

自动化：

- 任务策略
- 钉钉推送
- 问答 Agent

管理：

- 账号权限
- 安全白名单
- 系统配置

#### 6.5.2 总览

展示：

- 历史候选人
- 今日新增
- 昨日新增
- 推荐候选人
- 平均匹配度
- 报告数量
- 数据来源分布
- 账号分布
- 高匹配推荐 Top 10

#### 6.5.3 候选人模块

视图：

- 表格列表
- 看板视图
- 岗位视图

表格能力：

- 筛选
- 排序
- 按账号筛选
- 按岗位筛选
- 按状态筛选
- 导出候选人
- 导出推荐
- 查看详情

#### 6.5.4 岗位库模块

能力：

- JD 列表。
- JD 编辑。
- JD 删除。
- 待补录列表。
- 重新匹配候选人。
- 查看匹配到的候选人信息。
- 重复岗位合并。

#### 6.5.5 账号权限模块

能力：

- 用户首次登录后选择或绑定招聘账号。
- 管理员审核绑定关系。
- 管理员可新增、编辑、停用用户。
- 管理员可配置用户可见账号。

#### 6.5.6 自动化策略模块

按账号配置：

- 工作日。
- 工作时间段。
- 随机请求间隔。
- 详情停留时间。
- 滚动模式。
- 每日操作上限。
- 每小时操作上限。
- 单轮候选人数。
- 长暂停频率。
- 搜索关键词分散池。

#### 6.5.7 系统配置模块

能力：

- 后端地址配置。
- 插件安装包下载。
- 插件版本更新提示。
- 自动生成带后端地址的插件包。
- 数据保存策略。
- 原始简历全文保存开关。
- 展示脱敏策略。

### 6.6 钉钉机器人

#### 6.6.1 定时推送

- 默认推送“上次推送后到本次推送前”的增量数据。
- 推送正文包含新增候选人、收到简历数量、推荐候选人、数据来源。
- 推送 Excel 推荐表。
- 文件直发失败时，自动补发可下载入口。

#### 6.6.2 Excel 字段

字段：

- 姓名
- 是否有附件简历
- 数据来源
- 账号
- 岗位
- 匹配度
- 学历
- 推荐等级
- 求简历状态
- 最后沟通时间
- 候选人详情摘要

#### 6.6.3 问答

支持问题类型：

- 招聘数据统计。
- 候选人检索。
- 推荐或不推荐原因分析。
- 运营建议。

钉钉群里任何人提问，默认可回答全部数据。

### 6.7 权限与隐私

#### 6.7.1 登录体系

- Web 后台必须有登录体系。
- 管理员和普通用户权限隔离。
- 普通用户仅能查看审核通过的绑定账号数据。

#### 6.7.2 数据保存

- 默认永久保存历史数据。
- 管理员可按账号和时间范围清理数据。
- 管理员可配置是否保存原始简历全文。

#### 6.7.3 敏感信息

- 存储时可保留原文。
- 展示时按权限脱敏。
- 手机号、邮箱、微信号等联系方式必须自动脱敏。
- 管理员可查看完整内容。
- 普通用户按权限查看脱敏内容。

### 6.8 操作证据

自动点击证据保存：

- 时间。
- 动作类型。
- 候选人。
- 岗位。
- 点击按钮文本。
- 页面 URL。
- 动作结果。

不保存默认截图，避免增加存储和隐私风险。

## 7. 非功能需求

### 7.1 可用性

- 插件任务必须可暂停。
- 后台任务状态必须可见。
- 关键失败原因必须可读。
- 后台页面必须按功能分区，避免所有配置堆叠。

### 7.2 性能

- 候选人列表查询默认分页或限制条数。
- 大模型调用超时可配置。
- Excel 导出必须在合理时间内完成。

### 7.3 安全

- 支持 Host 白名单。
- 支持 Origin 白名单。
- 支持客户端 IP 白名单。
- 支持接口频率限制。
- 支持内网部署。

### 7.4 可维护性

- 插件、Web 后台、后端服务分层清晰。
- JD 抽取、候选人抽取、匹配评分、推送逻辑应独立模块化。
- PRD、部署文档、接口清单和验收清单需随版本维护。

## 12. 验收标准

### 12.1 候选人采集

- 当前列表全部可见候选人均被尝试处理。
- 不满足最低入库标准的记录不入库。
- 入库记录姓名不得为空。
- 采集内容不得混入其他候选人信息。

### 12.2 岗位要求库

- 新岗位首次出现时必须尝试抓取 JD。
- 抓取成功后岗位要求库出现该岗位。
- 抓取失败后岗位显示待补录。
- 重复岗位只保留一条有效 JD。

### 12.3 匹配与索要简历

- 40% 以上候选人必须索要简历。
- 找不到求简历按钮时必须发送兜底消息或记录失败原因。
- 已收到附件简历候选人不得重复索要。

### 12.4 Web 后台

- 各模块只显示本模块相关内容。
- 账号筛选对候选人、推荐、岗位、统计同时生效。
- 岗位匹配后可查看匹配候选人详情。

### 12.5 钉钉推送

- 定时推送按增量范围生成。
- 正文格式符合配置。
- Excel 字段符合 PRD。
- 文件直发失败时有可下载兜底入口。

### 12.6 权限

- 管理员可见全部数据。
- 普通用户只可见授权账号数据。
- 未审核用户不可查看业务数据。
